# วางบนเซิร์ฟเวอร์ในเครือข่าย MWA เพื่อใช้ค่าสด (Real-time)

## ทำไมต้องวางบนเซิร์ฟเวอร์นี้
เว็บบน GitHub Pages เป็น **HTTPS** แต่ endpoint ของ SCADA เป็น **HTTP** (`http://172.16.193.162/...`)
เบราว์เซอร์จะบล็อกการเรียกข้าม (mixed-content) และ endpoint ไม่ได้เปิด CORS
→ **ค่าสดจะทำงานก็ต่อเมื่อหน้าเว็บถูกเสิร์ฟจาก origin เดียวกับ endpoint** คือเครื่อง `172.16.193.162` ผ่าน `http://`

## ไฟล์ที่ต้องใช้ (ชุดเดียว พอ)
- **`index.html`** — ตัวเว็บทั้งหมด (self-contained ~530 KB ไม่ต้องมีไฟล์อื่น)
- (ถ้าต้องการ) `presentation.html` — สไลด์นำเสนอ

> `master-data.json` เป็นแค่ไฟล์ต้นฉบับข้อมูล — ไม่จำเป็นตอนรัน เพราะ `index.html` ฝังข้อมูลไว้ในตัวแล้ว

## ขั้นตอนวาง (สำหรับผู้ดูแล Smart Map / IIS)
1. สร้างโฟลเดอร์บนเว็บรูทของ IIS เครื่อง `172.16.193.162` เช่น `C:\inetpub\wwwroot\smartmap\km\`
   (หรือโฟลเดอร์ใดก็ได้ที่อยู่ใต้โฮสต์ `172.16.193.162` ผ่าน http)
2. คัดลอก `index.html` ไปวางในโฟลเดอร์นั้น
3. เปิดใช้งานผ่าน: **`http://172.16.193.162/smartmap/km/`**

เท่านี้ค่าสดจะขึ้นครบทันที (System Map, Overview, Pumping Stations, RTU Monitoring) เพราะหน้าเว็บกับ endpoint อยู่ origin เดียวกัน — ไม่ติด mixed-content และไม่ติด CORS โดยไม่ต้องแก้โค้ดใด ๆ

### endpoint ที่หน้าเว็บเรียก (มีอยู่แล้วบนเครื่องนี้)
```
/smartmap/data_station_real.php      (ทุก 5 วิ — สถานี 10 แห่ง)
/smartmap/data_bk_real.php           (บางเขน/อุโมงค์ TR)
/smartmap/data_rcv_real.php          (วาล์ว RCV)
/smartmap/data_ss_real.php           (สามเสน) · /smartmap/data_tb_real.php (ธนบุรี)
/smartmap/rtu_query2.php             (สถานะ Alarm ของ RTU)
```
โค้ดใช้ URL เต็ม `http://172.16.193.162/smartmap/...` อยู่แล้ว จึงวางที่ path ไหนใต้โฮสต์นี้ก็ได้

## ถ้าอยากวางบนเครื่องอื่นในวง (ไม่ใช่ 172.16.193.162)
ต้องให้ผู้ดูแลเพิ่ม HTTP header ที่ไฟล์ PHP บน Smart Map:
```
Access-Control-Allow-Origin: *
```
แล้วเสิร์ฟ `index.html` ผ่าน **http://** (ไม่ใช่ https) ในวง — จึงจะเรียกข้ามเครื่องได้

## พฤติกรรมเมื่อเปิดนอกเครือข่าย (เช่น GitHub Pages)
- แสดง **Snapshot** (ค่าคงที่ ณ เวลาที่บันทึกไว้ในไฟล์ ระบุวันเวลากำกับ) ทุกหน้ามีป้าย `◆ Snapshot`
- ไม่มีการดึงค่าสด และไม่กระทบระบบภายใน
- อัปเดต Snapshot ให้ใหม่ได้โดยเปิดหน้านี้จากในวงครั้งหนึ่งแล้วให้ผู้ดูแล refresh ค่า (หรือแจ้งผมให้ดึงชุดใหม่มาฝัง)

## ตรวจว่าค่าสดทำงานไหม
เปิดหน้าเว็บ → กด `F12` → แท็บ Console
- ถ้า **ไม่มี** error `Mixed Content` / `CORS` / `Failed to fetch` = เรียก endpoint ได้ ค่าสดจะขึ้น (ป้ายเปลี่ยนเป็น `● สด`)
- ถ้ายังเห็น error เหล่านั้น = ยังเปิดจาก origin ที่ผิด (เช่น https GitHub Pages หรือ file://)
